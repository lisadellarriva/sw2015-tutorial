So you got different results when running the reasoner compared to when not running it?

We can explain that! Click to show the explanation!

Well, what the reasoner basicly do is using the information already defined by the given ontology (i.e, relations, necessary/sufficient conditions, and all kinds of rules one can set for entities and properties) in order to infer new information (for examples: instances' classification; relations between classes and/or properties) and so add to what is already built.

Let's give a ilustrative example.

Imagine you created a new hot sandwich formed by white bread, hamburguer and spicy sauce.

From that, what is defined in the ontology is that there is a sandwich whose bread type is white, sauce flavor is spicy, has meat and its type is hamburguer and is hot. At this point, if you query for all sandwiches of type Hamburguer, this new one won't be part of the results! That is because the ontology doesn't know that this new sandwich not only match the class Sandwich, but also the classes such as Hamburguer, hotSandwich and a spicySandwich. That's what the reasoner is going to do!

When you run the reasoner, it will analyse the values of this new sandwich's properties, cross with classes' rules and add those new classifications for this new sandwich. Now, the ontology now that this new sandwich is also a hamburguer. So, if you now query for all hamburguers, this new one will be indeed be part of the results!
